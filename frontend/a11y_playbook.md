# Accessibility (a11y) & UX Playbook

## Overview
This playbook enforces WCAG compliance, screen reader support, and keyboard navigation across all UI components generated by the AI Agents.

## 1. Web Accessibility Audit

### Prompt Template
```markdown
Context:
- Target UI Component: {attached}

Task:
1. Audit the component for WCAG 2.1 AA compliance.
2. Ensure all interactive elements have appropriate `aria-labels` and `roles`.
3. Verify keyboard focus management (tab order and visual focus rings).
4. Refactor the code to fix any identified accessibility violations without breaking the visual design.
```

## 2. Mobile Accessibility (Android/iOS)

### Prompt Template
```markdown
Task:
1. Audit the View layer for proper semantic grouping (TalkBack on Android / VoiceOver on iOS).
2. Ensure touch targets are at least 48x48dp.
3. Validate that color contrast ratios meet the 4.5:1 requirement for text.
```
